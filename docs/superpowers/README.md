# Historical designs & plans

These `plans/` and `specs/` are **implementation archaeology** — useful for
“why did we try X?”, not for “what ships today?”.

**Authoritative current architecture:** repo-root `AGENTS.md`.

Do **not** implement from these files without reading live code. Later plans
supersede earlier ones. In particular, older specs that claim “no unit tests”
are obsolete — there is a JUnit suite under `src/test/kotlin/`.
